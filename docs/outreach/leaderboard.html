What We Test

Adversarial verification, not linting.

Traditional code quality tools check syntax. We check whether the application actually does what it's supposed to do.

Traditional Tools

ESLint, SonarQube, CodeClimate
  • - Syntax correctness and style enforcement
  • - Known vulnerability patterns (CVEs)
  • - Code complexity metrics
  • x Cannot detect empty handlers behind real buttons
  • x Cannot detect fake analytics dashboards
  • x Cannot detect unprotected admin routes
  • x Cannot detect IDOR vulnerabilities in business logic
  • x Cannot detect claims-vs-reality gaps

LUCID Verification

Formal claim extraction + codebase verification
  • + Extracts 80-150 testable claims per application
  • + Verifies every button, form, and API actually works
  • + Detects scaffolded UI with no backend
  • + Catches IDOR, broken auth, missing validation
  • + Identifies fake data and hardcoded responses
  • + Maps claimed features to actual implementations
  • + Scores security, functionality, architecture, compliance
  • + Produces actionable remediation tasks
Methodology

How we score platforms.

All benchmarked applications are user-built projects from public repositories or submitted codebases. No cherry-picked examples.

Claim Extraction

LUCID generates a formal specification for each application, extracting 80-150 testable claims across security, functionality, data handling, performance, and compliance.

Codebase Verification

Each claim is verified against the actual source code. Every function, route, handler, and data flow is traced. PASS, PARTIAL, FAIL, or N/A verdict per claim.

Scoring

Composite score (0-100) weighted across categories: security (30%), functionality (30%), architecture (20%), and compliance (20%). Severity-weighted to penalize critical gaps.
